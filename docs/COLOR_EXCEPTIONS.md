## 登記規範

### 1. 填寫欄位說明

- **檔案路徑**：例外發生的檔案路徑（相對專案根目錄）
- **位置**：具體的 CSS Class 或 HTML 元素
- **顏色值**：寫死的顏色代碼（如 `#FF5500`）
- **原因**：為什麼必須寫死？（品牌色、設計要求、系統限制等）
- **登記日期**：建立此例外的日期
- **負責人**：負責維護此例外的人
- **狀態**：
  - `Active`：正在使用
  - `Review`：需定期審查
  - `Deprecated`：已移除或過期

## 定期審查機制

### 每月審查清單

1.  檢查所有 `Active` 狀態的例外
2.  確認是否仍需要寫死顏色
3.  如可改為系統變數，則標記為 `Deprecated` 並修改程式碼
4.  更新登記表的「最後更新」日期

### 審查流程

```
1. 開啟 COLOR_EXCEPTIONS.md
2. 檢查所有 Active 狀態的例外
3. 評估是否需要維持寫死
4. 如需修改，更新程式碼並標記為 Deprecated
5. 更新登記表狀態
```
